## Agents

### 1. Researcher Agent

**File:** `.claude/agents/researcher.md`
**Model:** Sonnet
**Tools:** web_search, web_fetch, bash

**Job:**
- Find Thai government, university, FAO, and international sources
- HTTP-verify every URL before returning
- Reject any URL not returning 200
- Return ranked list with confidence levels

**Quality bar:**
- Minimum 6 Thai sources
- Minimum 3 international sources
- Minimum 4 high-confidence sources

### 2. Drafter Agent

**File:** `.claude/agents/drafter.md`
**Model:** Sonnet
**Tools:** view, create_file, str_replace, bash

**Job:**
- Write MDX following 13-section template
- Cite every claim with source from Researcher's list
- Apply confidence labels per section
- Add Thailand applicability notes
- Apply safety language (no chemical dosages, etc.)
- Self-validate before output

**Output:** `src/content/crops/{slug}.mdx`

### 3. URL Verifier (Script)

**File:** `scripts/verify-urls.sh`
**Type:** Bash script (NOT AI — for reliability)

**Job:**
- Extract every URL from MDX file
- HTTP HEAD request each (10s timeout, follow redirects)
- Accept: 200, 301, 302
- Reject: 404, 500, timeout, etc.

**Why script not AI:** URL checking is deterministic. Using AI introduces unnecessary variability and risk of false-positive approval.

### 4. Content Verifier Agent

**File:** `.claude/agents/content-verifier.md`
**Model:** Sonnet
**Tools:** view, web_fetch, bash, str_replace

**Job:**
- Re-fetch every cited source (fresh, no Drafter context)
- Compare claims in MDX vs actual source content
- Flag hallucinations, misattributions, distorted translations
- Check SOURCE_POLICY compliance (15-word quote rule, etc.)
- Check SAFETY_POLICY compliance (no chemical dosages, etc.)
- Auto-fix MEDIUM issues (max 1 retry)
- BLOCK on any 🔴 BLOCKER issue

**Critical:** Operates in **fresh context** — no shared memory with Drafter to prevent correlated errors.

### 5. Decision Agent

**File:** `.claude/agents/decision.md`
**Model:** Sonnet
**Tools:** bash, view, create_file, str_replace

**Job:**
- Orchestrate the 4 stages above
- Apply decision logic at each gate
- Auto-commit + push if all pass
- Log to AUDIT_LOG.md
- Halt and log to PIPELINE_FAILURES.md if any blocker

## Slash Commands

### `/add-crop <name>`
Adds a single crop via full pipeline.

Example:
```
/add-crop กะเพรา
/add-crop holy basil

```

## Safety Mechanisms

### 1. URL Verification (Two-Stage)
- Researcher verifies URLs at discovery time
- URL Verifier script re-checks before publication
- Content Verifier re-fetches once more

**Reasoning:** URLs can change, redirect, or fail between agents. Three checks catches drift.

### 2. Fresh-Context Verification
- Content Verifier runs in subagent with NO memory of Drafter
- Prevents correlated hallucinations
- Re-reads policies from disk every run

### 3. Auto-Fix Limit
- Maximum 1 retry per content piece
- Only 🟡 MEDIUM issues are auto-fixable
- 🔴 BLOCKER issues halt immediately, no auto-fix attempt

### 4. Rate Limiting
- Maximum 5 pipeline runs per hour
- Maximum 50 crops per day
- Prevents runaway loops or accidental scaling

### 5. Emergency Halt
```bash
touch .claude/HALT
```
Pipeline checks before each run. Resume by deleting the file.

### 6. Logging
- All runs logged to `docs/AUDIT_LOG.md`
- All failures logged to `docs/PIPELINE_FAILURES.md`
- Verifier statistics in `.claude/logs/verifier-stats.json`

## What Could Still Go Wrong

This pipeline reduces but does not eliminate risk:

| Risk | Mitigation | Residual |
|---|---|---|
| Hallucinated URLs | URL Verifier (script) + re-check | Low |
| Misattributed citations | Content Verifier re-fetch | Medium |
| Subtle factual errors | Verifier judgment (Sonnet) | Medium |
| Outdated info (knowledge cutoff) | Source recency check | Medium |
| Source goes offline post-publish | `/audit-recent` periodic re-check | Low |
| Drafter + Verifier shared blind spots | Fresh context per agent | Medium |
| New policy violations not in checklist | Manual prompt updates | Low |

**Acceptable for current operating mode** per AUDIT_LOG entry 2026-04-29.

## When to Halt and Re-evaluate

Trigger pipeline review if:
- Failure rate > 20% over 10 runs
- Multiple readers report errors via GitHub Issues
- A specific source is found to have been mis-cited multiple times
- Content Verifier acceptance rate > 95% (too lenient) or < 75% (too strict)

Review checklist:
1. Read `docs/PIPELINE_FAILURES.md`
2. Run `/audit-recent 20`
3. Identify pattern
4. Update relevant agent prompt
5. Test on 3 crops before resuming
6. Document changes in AUDIT_LOG

---

## Reverting to Human-Review Mode

If you want to revert the policy override:

1. Touch `.claude/HALT` to stop pipeline
2. Append AUDIT_LOG entry: "Reverted to Definition A (human review)"
3. Update `docs/SAFETY_POLICY.md` Section 4 back to original
4. Update `CLAUDE.md` Section 7 back to original
5. Update `README.md` to remove "Content Generation Notice"
6. Resume manual content production

No content needs to be removed — past auto-published crops remain valid under their AUDIT_LOG entries.
